# Project Update Memo

We should probably continue the current customer onboarding project because it is important for the company and the team has already spent a lot of time on it.

The latest version is almost ready, but there are still some problems. Sales wants more custom fields. Operations is worried that the workflow will be too hard to maintain. Engineering thinks the deadline is possible, but only if nothing else changes.

The project may improve activation by 15% to 25%, although this is only a rough estimate. We have not yet aligned on the exact measurement window. The current launch date is next month, but the team still needs to confirm dependencies.

There are several options:

1. Launch now and fix issues later.
2. Delay the project by two weeks.
3. Reduce scope, but this may disappoint sales.

I think option 3 is probably best, but we need to discuss it.
